# assessment frontend module

Purpose:

* owns Phase 34 assessor workspace screens for evidence package creation, package review, findings tracking, and assessor notes
* owns Phase 35 control mapping and control coverage screens

Public surface:

* `index.ts`

Dependencies:

* `shared/`
* `users/public.ts` for project list lookups

Notes:

* pages stay thin and delegate to assessment hooks and API functions
* evidence references are read-only snapshots returned by the backend package builder
* control mapping UI manages catalog items and mapping transitions through assessment hooks only
* control coverage UI is read-only and consumes coverage snapshots generated by the backend
